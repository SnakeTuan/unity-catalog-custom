# UC + MinIO Gateway: API Setup Guide

Step-by-step API calls to set up Unity Catalog with MinIO Gateway STS credential vending, from zero to running Spark jobs.

**Prerequisites:**
- UC server running on `localhost:8080` with `server.authorization=enable`
- MinIO gateway running on `localhost:9000`
- `server.properties` configured with `s3.minioGateway.0=true` (see [minio-gateway-sts-solution.md](./minio-gateway-sts-solution.md))

**Variables used throughout:**

```bash
UC=http://localhost:8080
BUCKET=dataplatform-dev-hn1
CATALOG=localtest_1
SCHEMA=default
USER_EMAIL=snaketuan@gmail.com

# Admin token — generated by UC server at startup
ADMIN_TOKEN=$(cat etc/conf/token.txt)
```

---

## Step 1: Get Admin (SERVICE) Token

The admin token is auto-generated at server startup and written to `etc/conf/token.txt`. This is a SERVICE token that can manage all resources and impersonate users.

```bash
ADMIN_TOKEN=$(cat etc/conf/token.txt)

# Verify it works
curl -s "$UC/api/2.1/unity-catalog/catalogs" \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq .
```

---

## Step 2: Create User

Uses SCIM2 API. The user needs to exist before you can impersonate or grant permissions.

```bash
curl -s "$UC/api/1.0/unity-control/scim2/Users" \
  -X POST \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/scim+json" \
  -d '{
    "displayName": "Tuan TM",
    "emails": [{"value": "'"$USER_EMAIL"'", "primary": true}],
    "active": true
  }' | jq .
```

**Check: List users**

```bash
curl -s "$UC/api/1.0/unity-control/scim2/Users" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/scim+json" | jq .
```

---

## Step 3: Get User's ACCESS Token (Impersonate)

Exchange the admin SERVICE token for a user-scoped ACCESS token. Use this token for all subsequent user operations and for Spark.

```bash
USER_TOKEN=$(curl -s "$UC/api/1.0/unity-control/auth/impersonate?user_email=$USER_EMAIL" \
  -X POST \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq -r '.access_token')

echo $USER_TOKEN
```

---

## Step 4: Create Storage Credential (Admin)

Create a credential for the MinIO gateway. The `role_arn` is a dummy value — MinIO ignores it, but UC requires it for the AwsIamRole type.

```bash
curl -s "$UC/api/2.1/unity-catalog/credentials" \
  -X POST \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "minio-gateway-cred",
    "purpose": "STORAGE",
    "aws_iam_role": {
      "role_arn": "arn:aws:iam::minio:role/minio-gateway"
    },
    "comment": "Dummy credential for MinIO gateway auth checks"
  }' | jq .
```

**Check: List credentials**

```bash
curl -s "$UC/api/2.1/unity-catalog/credentials" \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq .
```

---

## Step 5: Create External Location (Admin)

Maps an S3 path to the credential. This is used for authorization checks — UC verifies the user has permission on the external location before vending credentials.

```bash
curl -s "$UC/api/2.1/unity-catalog/external-locations" \
  -X POST \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "minio-gateway-location",
    "url": "s3://'"$BUCKET"'/tuantm/localtest_1",
    "credential_name": "minio-gateway-cred",
    "comment": "MinIO gateway external location for localtest_1"
  }' | jq .
```

**Check: List external locations**

```bash
curl -s "$UC/api/2.1/unity-catalog/external-locations" \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq .
```

---

## Step 6: Create Catalog with Managed Storage Root (Admin)

The `storage_root` tells UC where to store managed table data. It must fall under a registered external location.

```bash
curl -s "$UC/api/2.1/unity-catalog/catalogs" \
  -X POST \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "'"$CATALOG"'",
    "storage_root": "s3://'"$BUCKET"'/tuantm/localtest_1/managed"
  }' | jq .
```

**Check: List catalogs**

```bash
curl -s "$UC/api/2.1/unity-catalog/catalogs" \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq .
```

---

## Step 7: Grant Permissions to User (Admin)

The user needs several permissions to work with the catalog:

### 7a. Grant USE_CATALOG + CREATE_SCHEMA on catalog

```bash
curl -s "$UC/api/2.1/unity-catalog/permissions/catalog/$CATALOG" \
  -X PATCH \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "changes": [{
      "principal": "'"$USER_EMAIL"'",
      "add": ["USE_CATALOG", "CREATE_SCHEMA"]
    }]
  }' | jq .
```

### 7b. Grant CREATE_EXTERNAL_TABLE on external location

Required for creating external tables (tables with explicit `LOCATION`) under this path.

```bash
curl -s "$UC/api/2.1/unity-catalog/permissions/external_location/minio-gateway-location" \
  -X PATCH \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "changes": [{
      "principal": "'"$USER_EMAIL"'",
      "add": ["CREATE_EXTERNAL_TABLE"]
    }]
  }' | jq .
```

### 7c. (Optional) Grant CREATE_MANAGED_STORAGE on external location

Only needed if the user needs to create catalogs or schemas with custom `storage_root`. Not needed if admin already created the catalog/schema.

```bash
curl -s "$UC/api/2.1/unity-catalog/permissions/external_location/minio-gateway-location" \
  -X PATCH \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "changes": [{
      "principal": "'"$USER_EMAIL"'",
      "add": ["CREATE_MANAGED_STORAGE"]
    }]
  }' | jq .
```

**Check: Verify user's permissions**

```bash
# On catalog
curl -s "$UC/api/2.1/unity-catalog/permissions/catalog/$CATALOG" \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq .

# On external location
curl -s "$UC/api/2.1/unity-catalog/permissions/external_location/minio-gateway-location" \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq .
```

---

## Step 8: Create Schema (User)

Now switch to the user token. The user creates a schema under the catalog.

```bash
curl -s "$UC/api/2.1/unity-catalog/schemas" \
  -X POST \
  -H "Authorization: Bearer $USER_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "'"$SCHEMA"'",
    "catalog_name": "'"$CATALOG"'"
  }' | jq .
```

**Check: List schemas**

```bash
curl -s "$UC/api/2.1/unity-catalog/schemas?catalog_name=$CATALOG" \
  -H "Authorization: Bearer $USER_TOKEN" | jq .
```

---

## Step 9: Grant Schema Permissions to User (Admin)

After schema creation, grant the user permissions to create tables and use the schema.

```bash
curl -s "$UC/api/2.1/unity-catalog/permissions/schema/$CATALOG.$SCHEMA" \
  -X PATCH \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "changes": [{
      "principal": "'"$USER_EMAIL"'",
      "add": ["USE_SCHEMA", "CREATE_TABLE"]
    }]
  }' | jq .
```

> **Note:** If the user created the schema, they are already the OWNER and have all permissions on it. This step is only needed if the admin created the schema instead.

**Check: Schema permissions**

```bash
curl -s "$UC/api/2.1/unity-catalog/permissions/schema/$CATALOG.$SCHEMA" \
  -H "Authorization: Bearer $ADMIN_TOKEN" | jq .
```

---

## Step 10: Run Spark Test

Now everything is set up. Run the Spark test with the user's token:

```bash
cd minio-gateway

spark-submit \
  --packages "io.unitycatalog:unitycatalog-spark_2.13:0.4.0,io.delta:delta-spark_2.13:4.0.1,org.apache.hadoop:hadoop-aws:3.4.0" \
  test-spark.py
```

Make sure `test-spark.py` uses the `USER_TOKEN` (not the admin SERVICE token).

**Check: List tables after Spark creates them**

```bash
curl -s "$UC/api/2.1/unity-catalog/tables?catalog_name=$CATALOG&schema_name=$SCHEMA" \
  -H "Authorization: Bearer $USER_TOKEN" | jq .
```

## Status Check APIs (Quick Reference)

All use `GET` with `Authorization: Bearer $TOKEN` header.

| What | Endpoint |
|------|----------|
| List users | `GET /api/1.0/unity-control/scim2/Users` |
| List catalogs | `GET /api/2.1/unity-catalog/catalogs` |
| List schemas | `GET /api/2.1/unity-catalog/schemas?catalog_name={catalog}` |
| List tables | `GET /api/2.1/unity-catalog/tables?catalog_name={catalog}&schema_name={schema}` |
| List credentials | `GET /api/2.1/unity-catalog/credentials` |
| List external locations | `GET /api/2.1/unity-catalog/external-locations` |
| Get catalog permissions | `GET /api/2.1/unity-catalog/permissions/catalog/{name}` |
| Get schema permissions | `GET /api/2.1/unity-catalog/permissions/schema/{catalog.schema}` |
| Get table permissions | `GET /api/2.1/unity-catalog/permissions/table/{catalog.schema.table}` |
| Get ext location permissions | `GET /api/2.1/unity-catalog/permissions/external_location/{name}` |
| Get credential permissions | `GET /api/2.1/unity-catalog/permissions/credential/{name}` |

---

## Permission Reference

| Permission | Granted On | Purpose |
|-----------|-----------|---------|
| `USE_CATALOG` | catalog | Access catalog contents |
| `CREATE_SCHEMA` | catalog | Create schemas in catalog |
| `USE_SCHEMA` | schema | Access schema contents |
| `CREATE_TABLE` | schema | Create tables in schema |
| `SELECT` | table | Read table data |
| `MODIFY` | table | Write table data |
| `CREATE_EXTERNAL_TABLE` | external location | Create external tables under this path |
| `CREATE_MANAGED_STORAGE` | external location | Use this path as managed storage root for catalogs/schemas |
| `READ_FILES` | external location | Read files (used for temp credential vending - reads) |
| `WRITE_FILES` | external location | Write files (used for temp credential vending - writes) |
| `OWNER` | any resource | Full control — bypasses all permission checks |

## Cleanup (Optional)

```bash
# Delete tables (Spark-created)
curl -s "$UC/api/2.1/unity-catalog/tables/$CATALOG.$SCHEMA.orders" \
  -X DELETE -H "Authorization: Bearer $ADMIN_TOKEN"
curl -s "$UC/api/2.1/unity-catalog/tables/$CATALOG.$SCHEMA.customers" \
  -X DELETE -H "Authorization: Bearer $ADMIN_TOKEN"

# Delete schema
curl -s "$UC/api/2.1/unity-catalog/schemas/$CATALOG.$SCHEMA" \
  -X DELETE -H "Authorization: Bearer $ADMIN_TOKEN"

# Delete catalog
curl -s "$UC/api/2.1/unity-catalog/catalogs/$CATALOG?force=true" \
  -X DELETE -H "Authorization: Bearer $ADMIN_TOKEN"

# Delete external location
curl -s "$UC/api/2.1/unity-catalog/external-locations/minio-gateway-location" \
  -X DELETE -H "Authorization: Bearer $ADMIN_TOKEN"

# Delete credential
curl -s "$UC/api/2.1/unity-catalog/credentials/minio-gateway-cred" \
  -X DELETE -H "Authorization: Bearer $ADMIN_TOKEN"
```
